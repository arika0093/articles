nbgvというかgitの使い方だけど

* v0.1.0をリリース済
* mainブランチにいくつも変更加えた
* release/v0.1 ブランチに反映して v0.1.1 としてリリースしたい
* そのままmergeするとv0.2変更のコミットが入り込んでしまう
* また、大量のコミットが一度に追加されるので 自動バージョンが v0.1.1にならない
* 0.1.1に反映したくないコミットがあると困る

一個ずつcherry-pickしてた、けどめんどい
良い方法を見つけたのでメモ

* git checkout release/v0.1
* git merge main
* git rebase -i
* 色々出てくるので以下のようにする
  * `Set Version to '0.2'` -> 先頭のpickをdに変更(取り込まない)
  * 2行目はそのまま
  * 3行目以降のpickを全てsに変更
  * この際、取り込みたくないコミットがあったらdにする
* 保存
* コミットメッセージ編集画面が出てくるので 先頭に Ver 0.1.1と追記
* 保存

これで完了。

でもcherry-pickのようにoriginal commit id を残したい… 要調査。
